GitSense Contexts: Snippets for Code Changes

By Terrence Chen on June 12, 2016

Like snippets and gists, which provides an easy way to share segments of code; GitSense contexts, provides an easy way to share parts of Git's history.  With GitSense contexts, you can control what parts of Git's history you would like the end user to see, analyze and search.

For example, if you wanted to show others, what is changing in your 2.0 release, you can create a GitSense context, that only includes relevant branches and file changes for that release.  And by having others view Git's history through that context, you can ensure the changes that they see, will always be relevant to the 2.0 release.

In a nutshell, GitSense contexts, makes it possible for you to put code changes into context for others.  And in this blog post, we'll highlight some of the ways you can use GitSense contexts, to build software better, together.

Support for Bitbucket, GitHub and GitLab

Need to put repos that are hosted on Bitbucket, GitHub and GitLab into context?  No problem.  You can easily combine repos from Bitbucket Cloud/Server, GitHub/GitHub Enterprise and GitLab Cloud/Server.  And in the near future, you'll be able to include repos from Visual Studio Team Services as well.

Work smarter and faster with GitSense contexts

Here's some of the ways you can use GitSense contexts, to help you and others, to work smarter and faster:

  • Create an active releases context.  With this context, you'll be able to quickly tell what's in release and what's not.
    https://gitsense.com/insight?b=gitlab:gitlab-org/gitlab-ce:maste
  • See what others are working on, with a teams context.  With this context, you won't have to sift through unnecessary noise, to see your teammates changes.
    https://gitsense.com/insight?q=author:rsc@golang.org+author:rob@golang.org
  • Create a Jira issues context.  With this context, you'll be able to show changes for certain Jira issues, which should come in handy for daily scrum meetings.
    https://gitsense.com/insight?q=author:rsc@golang.org+author:rob@golang.org
  • Create a forked context, to help keep track of repos that you forked from.
    https://gitsense.com/insight?q=author:rsc@golang.org+author:rob@golang.org

For more examples and to learn to how create and share your own contexts, make sure to checkout the Quick Introduction to GitSense Contexts Tutorial context.

Native support for Bitbucket Cloud is coming soon.  With the GitSense addon for Bitbucket cloud, you'll be able create private gitsensable repositories for yourself and others.
